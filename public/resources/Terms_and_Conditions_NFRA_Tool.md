## 18. DOCUMENT HASHING SPECIFIC TERMS

### 18.1 Hashing is Optional

Document hashing functionality is **OPTIONAL**.

**You are NOT required to:**
- Use the hashing feature
- Upload any documents (documents are NEVER uploaded)
- Store hashes in cloud (local-only storage available)

### 18.2 How Hashing Works

**IMPORTANT - UNDERSTANDING THE PROCESS:**

1. **Client-Side Processing:**
   - You select files from Your computer
   - Files are read in Your browser only (using FileReader API)
   - SHA-256 hash is generated in Your browser (using Web Crypto API)
   - Original files NEVER leave Your computer
   - Only the 64-character hash value is generated

2. **Hash Storage:**
   - Hash is stored in Your browser (IndexedDB) - always
   - Optionally, hash can be backed up to cloud (Firestore) - Your choice
   - Original document is NOT stored anywhere (not local, not cloud)

3. **Verification:**
   - You select the same file again
   - Hash is regenerated from the file
   - New hash is compared to stored hash
   - Match = file unchanged; Mismatch = file altered
